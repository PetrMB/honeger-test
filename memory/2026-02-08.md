### 21:40 - macOS Permissions Pro Peekaboo

Petr mi ukázal screenshot System Settings s povolenými aplikacemi pro Screen Recording. Myslel jsem že to znamená že bych měl vidět co se děje na obrazovce, ale peekaboo hlásilo "Screen Recording permission is required".

**Problém:** I když byly povolené `Clawdbot`, `node` a `Terminál`, samotný binary `peekaboo` tam nebyl!

**Řešení:**
1. Otevřel jsem System Settings → Screen Recording pomocí URL scheme
2. AppleScript automation: klikl na tlačítko "+", navigoval do `/opt/homebrew/bin/`
3. Vybral `peekaboo` binary a potvrdil
4. Ověřil že `peekaboo permissions` teď hlásí "Granted" ✅

**První screenshot pomocí peekaboo:**
```bash
peekaboo see --mode frontmost --annotate
```
Úspěch! 24 interactive elements detekováno, annotated screenshot vytvořen.

## Lekce

### 🎯 "Když to umíš, udělej si z toho skill"
- Není to jen o tom vyřešit problém TEĎ
- Je to o tom **znát to příště**
- Skills = procedurální paměť pro budoucnost
- Real-world testing je důležitý - teď vím že to opravdu funguje

### 🔧 macOS Permissions Pattern
- CLI tools často potřebují permissions které shell/Terminal už má
- Binary musí být explicitně v seznamu
- URL schemes jsou rychlý způsob jak otevřít správnou sekci
- AppleScript může automatizovat UI, ale cesty se mění mezi macOS verzemi

### 📝 Skill Design
- Frontmatter `description` je triggering mechanism - musí být jasný
- Progressive disclosure: SKILL.md = essentials, references/ = details
- Real-world example v dokumentaci pomáhá
