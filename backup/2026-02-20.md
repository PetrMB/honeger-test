## 🎓 YouTube Video Summary (AuofNgImNhk)

### Topic: **Making OpenClaw Actually Remember Things**  
**Speaker:** Ray Fernando  
**Key Concept:** QMD (Query-Driven Memory)

### Core Problem
- OpenClaw agent accumulates memory (daily logs, notes, docs) → context window bloat  
- Every prompt carries 2K–20K+ tokens of history  
- Memory compaction helps, but query efficiency still suffers

### QMD Solution
- **BM25** + **Vector Search** + **Reranking**  
- External memory lookup instead of storing everything in context  
- Query expansion → multi-phase search → context injection (top 3–5 snippets)

### Implementation Steps
1. `openclaw plugins install @openclaw/qmd`  
2. `openclaw qmd setup`  
3. Configure BM25 index + LanceDB vectors  
4. Automatic memory flush before compaction  

### Benefits
- Reduce context size from 20K → ~1K tokens  
- Faster, more relevant recall  
- Decouples memory growth from token budget
